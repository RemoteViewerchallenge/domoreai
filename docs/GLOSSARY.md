| Term | Definition |
| :--- | :--- |
| **Workspace** | The root container of the application. Currently transitioning to "Focus Workspace". |
| **SuperNode** | A visual representation of a file, folder, or agent. It can be "Frozen" (static) or "Thawed" (interactive). |
| **AiButton** | The "Micro-Kernel" button found in headers. Left-click to prompt, Right-click for the Power Menu. |
| **Focus Stack** | The layout engine that manages screen real estate. It ensures one "Main" task and one "Context" task. |
| **Cooperative Editor** | The TipTap-based document where Human and AI write together. The "Flight Deck" of the operation. |
| **OmniObserver** | The sidebar widget that streams logs, thoughts, and browser screenshots from background agents. |
| **Intervention** | The act of a human clicking a "Stuck" agent to manually enter a terminal command or fix a bug. |
| **Ghost Record** | A database entry (e.g., in `Role` table) that represents a file on disk. If the file is deleted, the record is "Ghosted". |
| **FlyonUI** | The mandated component library. Agents must prioritize this over custom Tailwind. |
| **The Stage** | The primary work area (66% of screen) where the active task is displayed in full detail. |
| **The Context** | The secondary reference area (33% of screen) showing supporting material side-by-side. |
| **The Ticker** | The top bar showing minimized background processes and task history as clickable chips. |
| **Constitution** | The immutable laws governing agent behavior and project structure. |
| **Code Rules** | Technical standards defining how code should be written and organized. |
| **Domain** | A project workspace with its own tech stack, agents, and indexed codebase. |
